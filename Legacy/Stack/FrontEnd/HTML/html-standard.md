# 웹 표준화의 시작

```
웹 초창기 시대에는 웹 어플리케이션에서 진행되는 거의 모든 것들을 서버 내에서 처리했습니다. 서버에서 사용자의 데이터를 처리하는 방식은 훌륭하지만, 브라우저와 같은 사용자 환경에서는 서버에서 처리할 수 없거나 그 문제에 대응할 경우 서버의 비용이 더 많이 발생하는 경우가 있어 Adobe사의 Flash와 같은 대체제를 사용하곤 했습니다.

시간이 흘러, 웹은 새로운 시대를 만나게 됩니다. 많은 변화가 있었지만, 대표적으로 두 가지를 언급하자면 첫째는 World Wide Consortium(W3C), Web Hypertext Application Technology Working Group(WHATWG)와 같은 웹을 위한 그룹들의 탄생, 둘째는 모바일 시장의 출현이라고 할 수 있습니다.

다양한 OS와 다양한 디바이스, 다양한 브라우저에서 초래된 이슈들을 Flash와 같은 대체제를 사용해서 해결하는 데에는 한계가 있어 브라우저 내에서 CSS와 스크립트를 통해 해결하려고 노력했습니다. 서버에서 처리되는 과정의 일부분을 브라우저에 위임하게 되면 서버에서 발생되는 비용과 리소스를 절감할 수 있었기 때문입니다. 하지만 브라우저 내에서 사용되는 기술들은 표준이 없어 개발과정은 생산적이지 못했습니다.

그러나 1994년 W3C라는 기구에서 웹의 표준화를 위해 움직이기 시작했고 2004년 웹을 발전시키고자 하는 사람들이 모여 WHATWG라는 그룹을 만들어 브라우저에서 보다 많은 일을 할 수 있도록 여러가지 기능들을 제시하기 시작했습니다. 
(WHATWG는 애플, 모질라 재단, 오페라 소프트웨어의 개인들이 설립한 그룹입니다.) 여러 노력 덕분에 웹표준의 시대가 열렸으며 현재도 계속 발전하고 있습니다.

HTML의 경우 2012년 HTML5라는 표준 스펙이 제시되어 2014년에 권장사항 채택되었고, 2017년 HTML5는 5.2라는 스펙까지 발전했습니다.
```
